（一）启动组件

使用idea工具打开，加载后可以看到rocketmq有许多子项目，很多同学一脸懵逼，从哪里入手呢？这时候我们可以结合官网来看。

通过Quick Start 这一章节可以看出，一个基础的rocketmq需要启动下面两个组件

  • namesrv （rocketmq自实现的服务治理，相当于zookeeper)
  • broker (rocketmq 节点)

先启动 namesrv 的 debug 模式，不出意外会出现一下报错

这是因为环境变量中没有设置rocketmq_home 变量，有两种方式解决：

  • 在环境变量中设置，类似于jdk的设置，设置的值是rocketmq 子项目 distribution 的目录
  • 直接在源码中添加环境变量
